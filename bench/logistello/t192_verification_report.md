# T192: Logistello book 検証資産化レポート

ステータス: **完了**。100局面の完全読み照合まで完走した(WLD一致率99%、

## 1. 形式確認(最初の関門)

`https://skatgame.net/mburo/logbook.wtb.gz` をダウンロード・gzip展開すると
`logbook.wtb`(2,564,228バイト)が得られる。ヘッダ16バイトを読むと:

| フィールド | 値 |
|---|---|
| 作成日 | 1999-12-05 |
| N1(ゲーム数) | 37709 |
| N2 | 0(ゲームアーカイブ) |
| 対象年 | 1999 |
| P1(盤サイズ) | 0(= 8x8) |
| P2(種別) | 0(通常) |
| P3(深さ) | 0 |

本体長 `2564228-16=2564212` バイトは `37709*68` と一致し、
`train::wthor::parse` の検証(N2=0, P1∈{0,8}, N1×68==本体長)をそのまま通った
(`cargo build --release -p train --bin logistello_extract` でのビルド・実行成功、
`cargo test -p train` 全件パスで確認)。**`train/src/wthor.rs` 本体の変更は不要だった。**

なお `P3(深さ)` が `0` であり、公式FFOアーカイブ(WTH_2023/2024.wtb、P3=24)とは異なる。
Logistelloのbook生成ツールはこのヘッダフィールドを埋めていないとみられ、
「24空きまでWLD検証済み」という保証は、このヘッダ値ではなくダウンロードページの
記載(`https://skatgame.net/mburo/log.html`)由来である。

## 2. `theoretical_score` の意味(実データでの確認)

37709ライン全件について `theoretical_score`(1バイト)と `black_disc_count`
(実測終局黒石数、1バイト)を比較した結果、**全件で完全一致**した
(`equal count 37709 / diff count 0`、値域は10〜60)。

これは以下を強く示唆する:

- `theoretical_score` は「黒の最終石数」(0..64スケール、`black_disc_count`と同一定義)であり、
  石差(diff、-64..64)でも「diff+64」エンコーディングでもない
  (後者ならこの値域・完全一致は起こりにくい)。
- 本bookの全ラインは、実際に打たれた対局がそのまま「理論値」と一致している。
  すなわち Logistello の自己対戦は 24空き以降を(ダウンロードページの主張どおり)
  一貫して最適継続しており、対局の最終結果がそのまま「24空き局面の完全読みスコア」
  として使える。

**仮説(§5で完全読み照合済み)**:
24空き局面(手番 `side_to_move`)における完全読みスコア(手番視点、石差)は

```
black_diff = 2 * theoretical_score - 64
expectedScoreSideToMove = black_diff            (side_to_move == black のとき)
                         = -black_diff           (side_to_move == white のとき)
```

**最終結論(§5の完全読み照合結果を踏まえて)**: この仮説は**符号(WLD)としては
高精度(99/100)で正しい**が、**厳密な石差(完全読みスコアそのもの)としては
正しくない場合がある(62/100のみ完全一致)**。理由はダウンロードページ自身の記載
(「win/loss/draw determination at 26-22 empty squares, exact score 1-2 ply later」)
と整合する: このbookは24空き時点でのWLD正しさは保証するが、そこから先の実際の対局が
「厳密な石差最適」まで保証されているわけではない(WLDを保たったまま石差で損な手を
指すことがある)。したがって `theoretical_score` から導く `expectedScoreSideToMove` は
**WLDオラクルとしては使えるが、石差の厳密オラクルとしては使えない**。詳細は§5。

### 5.3 原因分析

- **自前ソルバーの疑いは低い**: 既存のFFO終盤問題(#40-49)は全問正解が確立済みの
  回帰(README/設計書に記載の完了条件)であり、本タスクでも再検証はしていないが
  既知の事実として反証に使える。また本タスクのdraw局面19/19完全一致・
  black_win/black_lossでもWLD符号がほぼ完全に一致(99/100)している点は、
  探索・評価・符号変換のいずれにも系統的なバグがないことを強く示す
  (系統的な符号バグがあればWLD一致率がここまで高くならない)。
- **データ側の説明が妥当**: ダウンロードページ(`https://skatgame.net/mburo/log.html`)
  自身が「win/loss/draw determination at 26-22 empty squares, exact score 1-2 ply later」
  と明記しており、**このbookが24空き時点で保証しているのはWLDのみで、厳密な石差の
  最適性ではない**。つまり実際の自己対戦は、WLD(勝敗分岐)を保った範囲内で
  厳密には非最適な手を指すことがあり、それが`theoretical_score`(=対局の最終結果)と
  「24空き局面の真の完全読みスコア」との間に数石のズレを生む。不一致38件のうち37件は
  ±2〜±8(1〜4石分)の小さなズレに収まっており、この説明と整合する。
- **唯一の例外(`t192-logistello-025`、差分+14・WLDも反転)**は、このズレが単なる
  「わずかな非最適継続」では説明しきれない大きさであり、当該ライン単独で
  何らかのより大きな非最適継続(またはWLD境界に近い局面での深い読み違い、
  ただしそれは対局した側=Logistello側の問題であり本エンジン側の問題ではない)が
  あった可能性が高い。1/100の外れ値であり、全体傾向(WLD 99%一致)を覆すものではない。

### 5.4 結論(`theoretical_score`の意味、最終)

- `theoretical_score`(=`black_disc_count`と全件一致)は「黒の最終石数」であり、
  **24空き局面のWLD(手番視点の勝敗)を高信頼度(99/100)で与える独立オラクル**として
  使える。
- 一方、**石差(厳密スコア)の独立オラクルとしては使えない**(62/100のみ完全一致、
  残りは主に±2〜±8のズレ)。これはエンジン側の不具合ではなく、bookの出典が
  そもそも保証している範囲(WLDのみ)を超えて期待していたことによる。

## 6. 今後の用途(確定)

- **WLDオラクルとしての回帰テスト化は有望**: `expectedWldSideToMove`は99%の
  一致率を示しており、既存のFFO終盤問題・T096/T157オラクルセット(いずれも
  厳密スコアの一致を見る)とは異なる粒度(勝敗のみ)ではあるが、
  出自の異なる独立検証資産として`bench/`配下の定期ベンチに組み込む価値がある。
  1件の外れ値(`t192-logistello-025`)は既知の例外としてスキップ対象に記録して運用する。
- **石差の厳密回帰テストには不向き**: `expectedScoreSideToMove`をFFO/T096/T157と
  同格の「厳密スコア一致」ゲートに使うことは推奨しない(62%程度の一致率では
  偽陽性の失敗検知が多発する)。
- 具体的なCI組み込み(頻度・失敗時の扱い)は本タスクのスコープ外とし、
  必要になった時点で別タスクとして起票する。
