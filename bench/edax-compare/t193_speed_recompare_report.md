# T193: 対Edax速度比較の更新計測(高速化第2弾後、T180方式)

高速化シリーズ(T182 増分hash配線 / T184 `sort_by_cached_key` / T185 `ordered_moves`固定長配列化 / T186〜T191)適用後の現行HEADで、T180と同一手法・同一局面・同一Edax条件により対Edax速度倍率を**直接再測定**した。T184/T185は「Edax値を再測定せず、T180のEdax値に自エンジンの前後倍率を掛ける」相対倍率方式だったが、本タスクはユーザー依頼により**Edax側もこのセッションで直接実行**している。

## 結論(要約)

| 指標 | T180(基準) | T193(本タスク、実測) |
|---|---:|---:|
| 中盤深さ12、対Edax倍率(MPC off) | 56.8倍 | **約15.6倍** |
| 中盤深さ12、対Edax倍率(MPC on) | 68.5倍 | **約15.1倍** |
| 自エンジンNPS(MPC off) | 518,217 | 2,109,652(3回平均) |
| 自エンジンNPS(MPC on) | 429,832 | 1,944,520(3回平均) |
| EdaxNPS(同一20局面バッチ) | 29,426,082 | 32,870,844(off比較時3回平均)/29,309,012(on比較時3回平均) |

高速化シリーズにより自エンジンのNPSはT180比で約4.07倍(off)・4.52倍(on)に向上し、対Edax倍率はT180の57〜69倍から**約15〜16倍**まで縮小した。T184/T185時点の「相対倍率適用による推定値」(26〜30倍)よりもさらに縮んでいる。この差分は、(a)T186以降にも高速化(NPSさらに向上)が積み重なっていること、(b)本タスクでEdax自体を再実行した結果、EdaxのNPSがT180時点(29.4M)よりも本セッションでは平均29.3M〜32.9M(局面順・実行タイミングにより変動)とやや高めに出たこと、の両方が寄与していると考えられる。相対倍率方式で外挿した推定(26〜30倍)より縮み幅が大きく出たのは、外挿がT186以降の追加高速化を織り込んでいなかったためであり、本タスクの直接実測値がより正確な現在地である。

## 1. 計測条件(T180との同一性)

### 局面バッチ

`bench/edax-compare/t156_mpc_positions.json` から `emptyBucket=='29-36' and split=='test'` を `id` 昇順で抽出した先頭20件(`mpc-29-36-test-001`〜`mpc-29-36-test-020`)。T180/T182/T183/T184/T185と同一の局面集合(件数・IDが完全一致することをスクリプトのアサーションで確認)。

### 自エンジン側

- コマンド: `eval_cli best --depth 12 --exact-from-empties 0 --pattern-weights train/weights/pattern_v6.bin [--enable-mpc]`(T180と同一のコマンドテンプレート)
- ビルド元: HEAD `92341cacca22871de1d4f7135b80cb51968ad256`(T192のLogistello取り込みまで反映済み、高速化シリーズT182/T184/T185/T186〜T191をすべて含む)
- パターン重み: `train/weights/pattern_v6.bin`(sha256=`e69f3b1c33432bafd388af82adde64270ec8c51acaab070bbfff4cf4caf20fc9`、T183以降と同一)
- バイナリ2種を同一HEADから個別ビルド: off(featureフラグなし、sha256=`278c460e5d8179e58a82f7e2a1d97dbefd3799d3c421e27dc15d91dc6fccfaa0`)/on(`--features mpc_enabled`、sha256=`92cc5745f249196693bedaa47717f9fca54b0ed08804b84ca6db22d9c46bdf18`)
- **ノード数の既知値一致(要件3)**: 3ラウンドすべてでoff=**59,440,032**・on=**6,487,461**と完全一致(T180/T182/T183/T184/T185で確立済みの値と同一)。局面バッチ・重み・探索コードパスが過去タスクと同一であることの直接的な証拠。

### Edax側

- バイナリ: `bench/edax-compare/edax-extract/wEdax-x86-64.exe`(Edax v4.6公式リリース、`download-edax.ps1`取得、`.gitignore`対象で追跡外)
- **sha256 = `aabb5ac7d3f9a872fc0e7388ab1eee1d23c687f76c28642122524dc318b322b1`**
- オプション: `-solve <20局面をまとめた1つのOBFファイル> -l 12 -eval-file data/eval.dat -book-usage off -vv`(T180の記述「`-l 12`・book off」と一致)
- 集計方法: 20局面を1つのOBFファイルにまとめて1プロセスで解かせ、末尾の集計行`<file>: N nodes in H:MM.SSS (cpu = H:MM.SSS) (NPS nodes/s).`から総ノード数・総時間・NPSを抽出(T180と同じ「バッチ全体の集計行を使う」方式)。

**T180とのバイナリ同一性について**: T180のレポート/metaにはEdaxバイナリのsha256が記録されていなかった(本タスクで初めて記録した)。ただし、このファイル(`edax-extract/wEdax-x86-64.exe`)のタイムスタンプは2024-12-18(Edax配布物自体のビルド日時)のまま変化しておらず、`download-edax.ps1`の再実行や`edax-extract/`の作り直しがT022以降記録されていないことから、**T180で使われたものと同一のファイルであると判断できる**(ファイル自体が唯一のインスタンスであり複製・差し替えの経緯がない)。今後の比較のためにsha256を本レポートで確定させた。

## 2. 計測手順(交互3回・専有実行)

1. 開始前に`Get-Process`でpython/cargo/eval_cli/wEdax/rustc/node系プロセスの不在、`Get-CimInstance Win32_Processor`のLoadPercentageが1〜2%であることを確認(専有実行)。
2. MPC off比較・MPC on比較それぞれ独立に3ラウンド実施。各ラウンドは「自エンジン20局面バッチ」と「Edax 20局面バッチ(1プロセス)」の両方を実行し、ラウンドごとに実行順を入れ替える(round0: 自エンジン→Edax、round1: Edax→自エンジン、round2: 自エンジン→Edax)ことで、実行順序に起因する系統誤差(サーマルスロットリング・キャッシュ状態等)を避けた。
3. 自エンジン側のNPSは`eval_cli best`が返す`elapsedMs`(プロセス起動オーバーヘッドを含まない、エンジン内部計測)の20局面合計を分母に使用(T191等の既存手順を踏襲)。Edax側のNPSは`-vv`集計行が報告する値をそのまま使用(同様にEdax内部計測)。
4. 使用スクリプト: `t193_run_bench.py`(本タスク用に作成、リポジトリにはコミットせずセッションのscratchpadに保持)。

## 4. T180からの推移表

| 時点 | 対Edax倍率(MPC off) | 対Edax倍率(MPC on) | 測定方法 |
|---|---:|---:|---|
| T180(基準) | 56.8倍 | 68.5倍 | 直接実測(自エンジン・Edaxとも実行) |
| T184(sort_by_cached_key) | 約26.7倍 | 約29.6倍 | 相対倍率方式(Edax値はT180のまま据え置き、自エンジンのbefore/after倍率〈2.125x/2.313x〉をT180のEdax比に適用) |
| T185(ordered_moves残余最適化) | 約26.3倍 | 約28.7倍 | 同上(相対倍率方式のさらなる適用、参考値扱い) |
| **T193(本タスク、高速化第2弾後)** | **約15.6倍** | **約15.1倍** | **直接実測(自エンジン・Edaxとも本タスクで再実行)** |

T186〜T191で追加の高速化(NPS向上)が行われているため、T185からの直接的な倍率換算はできない(T185の「参考値」は相対倍率方式でありT186以降を反映していないため)。T193はT186〜T191を含む現行HEADでの直接実測であり、本タスク時点での正式な最新値である。

## 5. 終盤(endgame)側について

本シリーズ(T182/T184/T185/T186〜T191)は中盤探索(`negascout`/`ordered_moves`)のみを対象としており、終盤完全読みソルバー(`engine/src/endgame.rs`)は変更していない。したがって終盤の対Edax倍率は**T108のC2ベンチ実測(空き20-24・34局面・64MiB TT、無制限経路の幾何平均19.165672倍)を現行値として据え置く**(本タスクでの再計測は対象外、タスク仕様どおり)。

## 6. 検証・専有性

- 計測前後で`Get-Process`により競合プロセス不在を確認(python/cargo/eval_cli/wEdax/rustc/node系)。
- 自エンジンのノード数は6ラウンド(off3・on3)すべてで既知値と完全一致(`59,440,032`/`6,487,461`)、探索コードパス・局面・重みが過去タスクと同一であることを確認。
- 生データ: `bench/edax-compare/t193_speed_recompare_report.meta.json`(本レポートの数値の元になった全ラウンドの生JSON、sha256各種を含む)。
- 本タスクはコード変更を行っていない(計測のみ)。`cargo test`等の回帰確認は対象外(タスク仕様どおり)。
