# T063: 棋譜解析の評価グラフに軸の数値表示とカーソル追従ツールチップを追加

## 目的

ユーザー要望: 棋譜解析の評価グラフについて、(1)石数差予想を正しく反映したグラフになっているか確認する(T056/T059で内部の評価値自体は修正済みだが、グラフの見た目側の対応が必要)、(2)縦軸に評価値のスケール(数値目盛り)をきちんと表示する、(3)グラフ上でカーソルを合わせるとその時点の状況を簡単に把握できるようにする(現状はSVGネイティブの`<title>`によるツールチップのみで、表示が遅く見た目も素っ気ない)。

## 背景・コンテキスト(このリポジトリを知らない前提で読めるように)

- `app/src/analysis/EvalGraph.tsx`: 評価グラフ本体。T056で評価値自体は「累積ロス方式」(最善手は変化なし、悪手はロス分だけ悪化)に修正済み、T059で理論上限(±64石)へのクランプ・evalSource誤判定も修正済み。**内部データは既に正しい石数差スケールになっている**ため、本タスクは主にグラフの「見た目」(軸表示・インタラクション)の改善が中心。
- `EvalGraph.tsx`(48行目)の`CLAMP = 16`(石)は、`midgame/EvalBar.tsx`の`CLAMP`定数を踏襲した値で、グラフの縦軸表示範囲を±16石に固定している。T059で実際の評価値は理論上限±64石まで正しく取りうるようになったため、**この固定±16クランプが妥当か再検討すること**(大きな悪手が連続した場合、実際の評価値が±16を超えて描画上潰れて(頭打ちに)見える可能性がある)。
- 現状、グラフの縦軸には目盛り線・数値ラベルが一切無い(0石の基準線のみ)。ユーザーが「どの程度有利/不利なのか」を数値で読み取れない。
- 現状のツールチップ(153行目付近)は各点の`<circle>`要素に`<title>`要素を仕込むSVGネイティブの仕組みで、ホバーしてから表示されるまでのタイムラグが長く、スタイリングもできない。また、点そのものにカーソルを正確に合わせる必要があり、線上の任意の位置でホバーしても反応しない。

## 変更対象

- `app/src/analysis/EvalGraph.tsx` / `.css` — 以下を実装する:
  1. 縦軸に数値目盛り(グリッド線+ラベル)を追加する。表示範囲(クランプ値)は、実際のデータ範囲(理論上限±64石)を踏まえて見直す(固定±64にするか、実際の対局のデータ範囲に応じて自動調整するかは実装時に判断してよいが、いずれにせよ**軸の数値が実際の評価値と対応していること**、大きな評価値変動が不自然に頭打ちにならないことを確認する)。
  2. マウス/タッチでグラフ上をホバー(またはタップ)した際に、カーソルに追従する形で、その時点(最も近い手数)の情報(手数・手番・着手・評価値・ロス・分類)をまとめて表示するカスタムツールチップを実装する。SVGネイティブの`<title>`に頼らず、独自のツールチップ要素(SVG内の`<g>`+背景矩形+テキスト、またはHTML要素のオーバーレイ)を使い、素早く・見やすく表示されるようにする。カーソルがグラフ上のどの位置にあっても(点の上でなくても)、最も近い手のツールチップが表示されるようにする。

## 要件

1. 評価グラフの縦軸に、実際の石数差に対応した数値目盛りが表示されること。
2. 評価値が大きく変動する対局(悪手が連続する等)でも、グラフが不自然に頭打ちにならず、妥当な範囲で見やすく表示されること。
3. グラフ上でカーソルを動かすと、その位置に最も近い手の情報(手数・手番・着手・評価値・ロス・分類)が、素早く・見やすいツールチップで表示されること。ホバーした位置が正確に点の上でなくても反応すること。
4. 既存のクリックによる局面ジャンプ機能(`onSelectPly`)・悪手マーカーのクリック(`onMarkerClick`)は維持すること。
5. 375px幅等の狭い画面でも問題なく表示・操作できること(タッチ操作でのツールチップ表示も考慮すること)。
6. 既存のテストが壊れないこと。

## やらないこと(スコープ外)

- 評価値の計算ロジック自体(`analyzeGame.ts`)の変更は行わない(T056/T059で完了済み)。
- グラフのライブラリ化(Chart.js等の外部ライブラリ導入)は不要(既存のSVG直接描画方式を拡張する形で対応する)。
