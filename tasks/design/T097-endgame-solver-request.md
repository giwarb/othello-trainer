# T097 設計依頼: 終盤ソルバーの抜本強化(目標: Edax同等の終盤読み)

## 背景

ユーザー指示(2026-07-15):「終盤読みをEdaxと同等にしたい」。エンジン強化シリーズの次フェーズとして、評価関数(蒸留)より先に終盤ソルバーを強化する。

現状の実力差(既知の実測):
- 自作ソルバー: 空き18で3〜11秒、空き19で約17秒(native)。FFO #40-44は正解するが heavy 扱い(#45以降は時間がかかりすぎ ignore)。
- Edax: 同クラスの局面を概ね1秒未満〜数秒で完全読み(level相当で空き20-24を実用時間で解く)。
- 対局時の帰結: ノード予算160,000(決定性のためノード基準、壁時計1500msは保険のみ)の中でexact試行が完走できる空き数が浅く、T085aのexact quota(残予算60%)で空き19〜24は「データ収集対象・無理に上げない」扱いに留まっている。終盤の完全読み開始がEdaxより数手遅く、これが石差に直結している(T084分析: 空き19-24が最大弱点だった)。

## 現状の実装(調査して確認してほしい)

- `engine/src/endgame.rs` — 終盤ソルバー本体。既知の弱点(T085設計レポート指摘): 安定石カットなし、空き4〜1の専用処理なし、終盤専用ムーブオーダリングなし。
- `engine/src/search.rs` — exact切替(T085a: baseline-first、exact quota、AbortReason分離、親αβ窓の引継ぎ`solve_exact_window_limited_with_nodes`)、TTドメイン分離(Midgame/Exact)。
- `engine/src/tt.rs` — T086で品質保護済み(深いExact保持、両slot品質probe)。
- `engine/src/bitboard.rs` 等 — flip計算・mobility等の基礎。
- 制約: wasm32-unknown-unknown シングルスレッドが本番(WASM Threads不採用はユーザー確定方針)。ネイティブはベンチ・教師生成用。決定性(同一入力→同一結果)は絶対要件。FFO正解値の不変も絶対要件。
- ベンチ資産: `cargo test -p engine --release --test ffo_bench`(FFO #40-44 fast / heavy)、`bench/edax-compare/`(Edax実行体・vs_edax.py・t085/t096局面manifest・ノード予算regressionテスト)。

## 検討してほしい技術要素(Edaxが実装している既知の手法)

以下の採否・優先順位・段階分割を判断してほしい(これ以外に有効な手法があれば追加提案歓迎):

1. **安定石(stable discs)による早期カット** — 上下界の絞り込み。
2. **空きマスの事前リスト化+パリティ順序** — 象限パリティによるムーブオーダリング、空きリストの連結リスト管理。
3. **浅空き特化ソルバー** — 空き1/2/3/4の専用関数(Edaxの solve_1/2/3/4 相当)、盤面差分のみで石差計算。
4. **終盤用ムーブオーダリング** — 相手mobility最小化・隅隣接回避・TT move・ETC(enhanced transposition cutoff)。
5. **NWS(null window search)ベースの終盤PVS構造**。
6. **flip計算の高速化**(必要なら。SIMD/AVXはWASM制約に注意、wasm SIMD128の採否判断を含む)。
7. **TTの終盤特化利用**(既存Exactドメインの改善余地、置換規則はT086踏襲)。

## 判断してほしい点

1. **目標の定量化**: 「Edax同等」を検証可能な受け入れ基準に翻訳してほしい。候補: (a) FFO #40-49(または#40-59)の合計ノード数・壁時計を同一マシンのEdaxと比較して何倍以内なら合格か、(b) ノード予算160k内でexact完走できる空き数の中央値をどこまで上げるか、(c) 対局(60局 vs level 10)の平均石差への波及見込み。現実的な到達点(完全同等は無理なら何倍以内が妥当か)も率直に示してほしい。
2. **ノード削減 vs NPS向上の優先度**: 対局経路は決定性のためノード予算基準。ノード数を減らす施策(オーダリング・カット)が予算内の実効読み深さに直結し、NPSは壁時計保険の余裕にしか効かない。この構造を踏まえた優先順位。
3. **段階分割**: 1タスク=1ワーカー1セッションで完結し、各段階でFFO正解値不変+ノード数計測ができる分割。どの順に入れると計測が汚れないか(相互作用の分離)。
4. **exact切替ポリシーへの波及**: ソルバーが速くなった後、T085bのexact quota(60%)・切替空き数・`estimated_min_nodes`テーブルの再校正をどのタスクで行うか。
5. **リスク**: 決定性・FFO正解値を壊しやすい箇所(安定石カットの上下界バグ、パリティ順序とTT moveの相互作用等)と、各タスクのテスト戦略。
