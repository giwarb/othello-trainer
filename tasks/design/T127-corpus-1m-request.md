# T127設計依頼: 100万局面教師コーパスの設計(段階投資の第1段)

## 背景

- ユーザー承認(2026-07-17): 蒸留コーパスを100万局面へ拡大する(約2日の生成)。T126の切り分け実験で「同量ならEdaxラベルがWTHORラベルより約1石優位、これまでの蒸留の敗因は量」と確定し、v4×蒸留の学習曲線(45k=4.77→90k=3.63→180k=2.77、入れ子層化サブセット)の外挿は1Mで1.4〜1.63石(現本番v3×WTHOR=1.40)。
- 段階投資設計: 1Mの実測が約1.4以下なら次段の4M(WTHOR全局面級、約8日)を検討。1.7超なら打ち切り。
- 既存資産: `bench/edax-compare/gen_teacher_corpus.py`(T114で堅牢化済み: シャード/局面単位checkpoint・resume・--adopt-provenance・t096オラクル60キー除外・exactEmptiesThreshold=20・決定性-n 1)。expanded200k(200,000局面、閾値20世代)は完成・検証済み。
- **制約(T114で判明)**: 候補抽出は「1対局×1フェーズbin=1候補」設計で、WTHOR 2000-2024(74,024局)からのdedup後プールは**340,574件が上限**。expanded200kが200,000件を消費済み。100万には設計拡張が必要。

## 判断してほしい点

1. **候補プールの拡張方式**: (a)K拡張(1対局×1bin=K候補。T114検討時: K=3で理論約28万→2000-2024なら×3.7倍規模? 同一対局内の相関で実効サンプル数が目減りする懸念)、(b)年範囲拡張(WTHOR公式は1977年以降を公開。1977-1999の追加DL)、(c)両方、(d)「全局面サンプリング」への設計転換(WTHOR対局の全手局面からdedupして直接サンプル。最終目標の4M=全局面と一貫するが、45k-180k学習曲線の層化設計と前提が変わる)。**学習曲線の外挿の解釈可能性(1.4以下なら4M続行という判定)を守れる設計**を推奨してほしい。
2. **既存expanded200kの包含**: 新1Mセットが既存200kを部分集合として含む(入れ子)設計にできるか。できればEdax計算約10時間分(20%)を節約でき、曲線の入れ子性も保たれる。既存レコードの再利用方法(positionId/canonicalKeyの整合、シャード割当)の現実性を判断してほしい。
3. **フェーズbin配分**: 200kと同じ6bin均等(bin0の序盤は母集団制約あり)を1Mでも維持するか。序盤binは母集団が小さく頭打ちが強くなる(waterfall再配分の挙動確認)。
4. **X/C quota・opening cap・優先層(engineLoss)の扱い**: 200k設計のままスケールするか、1Mで変更すべき点はあるか。
5. **タスク分割案**: 生成(2日級、夜間)・検証・学習(v4×1M蒸留3seed)・判定を何タスクに割るか。生成中の並行作業可否(ユーザーの新規UI要望等が来た場合の調整ルール)。

## 制約
