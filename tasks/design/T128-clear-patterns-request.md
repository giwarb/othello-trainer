# 設計諮問: 中盤練習「明確な悪手パターン」の追加候補の検討(T128拡張)

## 背景

T128(実装中)で、中盤練習の悪手判定を「1手先の盤面対比で明確に言語化できるときだけ不合格」に作り替えている。初期セットは5パターン:

1. 相手の打てる場所の数の差(≥3)
2. 隅の献上(相手の合法手に隅が入る/入らない)
3. X/C打ち(空き隅隣接)
4. 壁・フロンティア石の増加差(≥4)
5. 確定石の損(差≥2)

ユーザー指摘(2026-07-18): 「この5つだと能力的に足りないシーンもある。もうちょっとわかりやすくて増やせそうなものがないか検討して」。つまり **カバレッジ(実際の悪手のうち説明できる割合)を上げたい。ただし「見た目上明らか・平易な日本語で言語化できる」ことが絶対条件**(わかりにくい説明を増やすくらいなら説明しない=合格扱いのまま、が方針)。

## 制約

- 深い探索を新たに要求しないこと(1手先の静的特徴+既存Workerコマンドの範囲。軽い追加計算=1手先局面での相手の合法手列挙・浅い静的量はよい)
- 既存資産とリンクすること: engine/src/explain.rs::compute_features(12特徴量)、app/src/analysis/motifs.ts(15モチーフ、未実装5種の理由コメントあり)、設計書 othello-trainer-design-verbalization.md の特徴量層・モチーフ一覧
- エンジン(Rust)の変更は「小さい追加特徴量1〜2個」までなら許容(コストとして明記すること)。ただし現在教師コーパス生成が走行中のため、engine変更を伴う案は「生成完走後に実装」の注記を付ける
- 初級〜中級者向けの学習アプリである(オセロクエスト帯)。上級概念でも「盤面2枚の対比+ハイライトで一目で分かる」なら可

## 判断してほしい点

1. **追加候補パターンの列挙と評価**: オセロの定石的な悪手概念(開放度・中割りの逆・辺の危険形(ウイング/山)・種石・一方的に打てる領域・連打の献上・罠(ストナー)等、これ以外も広く)から、(a)平易な言語化可能性 (b)静的検出可能性 (c)初中級者の実際の悪手での出現頻度(定性評価でよい) (d)既存特徴量/モチーフとの重複 の4軸で表にして評価する。
2. **推奨追加セット**: 上記から「追加する価値が高い順」に3〜6個を推奨し、各々の検出条件(擬似コード)・言語化テンプレ(平易な日本語)・閾値案・実装コスト(TSのみ/engine追加が要るか)を示す。
3. **カバレッジの見立て**: 5+追加パターンで「評価値上の悪手のうち説明できる割合」がどの程度になりそうか、根拠付きの定性見積もり(可能ならリポジトリ内の既存データ(棋譜解析キャッシュ・ベンチ棋譜等)を読み取り専用で数えて裏取り。無理なら理論的見積もりでよい)。
4. **段階導入の提案**: どれをT128の直後(T128b)に入れ、どれを後回しにすべきか。
