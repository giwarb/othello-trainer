# 設計諮問(セカンドオピニオン): 明確悪手パターン追加の設計レポート検証(T128b実装前)

## 背景

中盤練習の悪手判定を「1手先の盤面対比(悪手後 vs 最善手後、どちらも相手番)で明確に言語化できるときだけ不合格。説明できない差は合格扱い」に作り替えている(T128、実装進行中)。初期5パターン(相手の着手可能数差≥3 / 隅の献上 / X・C打ち / 壁・フロンティア差≥4 / 確定石差≥2)はタスク `tasks/T128-midgame-clear-blunder-compare.md` 参照。

追加パターンの検討を(Codex利用上限中だったため)Claudeエージェントに諮問し、レポート `tasks/design/T128-clear-patterns-report.md` を得た。これを踏まえオーケストレーターは第1波4種(隅の取り逃し/相手パスの取り逃し/自分の打てる場所の激減〔差≥3かつ絶対数≤4〕/石の取りすぎ〔フリップ差≥4・空き≥16ガード〕)を採用し `tasks/T128b-clear-patterns-wave2.md` を起票済み(未着手)。

## 依頼(判断してほしい点)

Codex(あなた)による独立検証として:

1. **レポートの評価の妥当性**: `tasks/design/T128-clear-patterns-request.md`(元依頼)と `tasks/design/T128-clear-patterns-report.md` を読み、採用4種・保留2種(ウィング・種石)・却下群(開放度・パリティ・ストナー等)の判断に誤り・見落としがないか。特に「初中級者の実際の悪手での出現頻度」の定性評価に異議があれば根拠付きで。
2. **検出条件・閾値の穴**: 4種の擬似コード・閾値(絶対数≤4、フリップ差≥4、空き≥16等)に、誤検出(明確でないのに発火)・見逃しの具体的な盤面反例がないか。オセロの戦術知識で検証すること。
3. **追加の見落とし候補**: レポートが挙げていない「平易に言語化でき、1手先の静的対比で検出できる」パターンで、追加価値が高いものがあれば提案(なければ「なし」と明言)。
4. **T128bタスク仕様の過不足**: `tasks/T128b-clear-patterns-wave2.md` の要件・受け入れ基準で実装者が誤解しうる点。

## 制約

- 読み取り専用調査。エンジン変更を要する提案は「生成完走後」の注記付きで。
- 方針の絶対条件は「見た目上明らか・平易な日本語」。捕捉率のための閾値緩和は方針違反。
