# T016: 定石データの調査・収集(公開情報から独自データベース化)

## 目的
定石練習モード(オセロクエスト式: 相手が定石内の手を重み付きランダムで選び、プレイヤーが定石を辿れるか練習する)で使う定石データを、**公開されている情報源から独自に調査・収集**して用意する。

## 背景・コンテキスト
- ユーザーからの明示的な方針(2026-07-08): オセロクエストのアプリ/データを直接複製することは避ける(他社データベースの無断複製を避けるため)。ただし「兎・牛・虎」などの定石名や手順自体は日本のオセロ界で古くから使われている共有知識であり、オセロクエスト固有の発明ではない。**公開されている定石名鑑・戦術解説サイト・書籍情報・Wikipedia等から、同じ定石体系(同じ名前・同じ手順)を独自に調査してデータベース化する**方針が承認されている。UX(相手が定石内の手を重み付きランダムで選ぶ、外れたら終了)はオセロクエスト式を再現してよい(これはゲームルールであり著作物ではない)。
- 設計書 `othello-trainer-design.md` §2.6「定石練習モード」を参照。データ構造は「局面ハッシュをキーにしたDAG(手順違いの合流を扱える)」「初手はf5に対称正規化」「主要定石100種前後(兎・虎・牛・鼠系とその派生)、深さ10〜25手」を想定しているが、**本タスクではデータ収集のみを行い、DAG構造・ハッシュ正規化・アプリへの組み込みは後続タスク(T017以降)で行う**。
- マス番号記法: `engine/src/bitboard.rs`の規約(`index = rank0*8 + file`, a1=0, h8=63)に合わせて、収集した手順は `a1`〜`h8` の標準記法(例: `f5d6c3d3c4...`)で記録する。

## 変更対象(新規作成)
- `bookgen/joseki-research.json`(または `.md` でも可。実装者が扱いやすい形式でよいが、後続タスクで機械的にパースできる構造化データにすること。リポジトリルート直下に `bookgen/` ディレクトリを新設。設計書§2.13のリポジトリ構成に対応)
- 出典を明記すること(URLまたは書籍名など、後から検証・追跡できる形で)

## 要件
1. WebSearch/WebFetchツールを使い、日本語のオセロ定石(初手f5から始まる進行で、黒4手目・白4手目あたりから分岐する体系)について、名前付きの定石(兎・牛・虎・レグホーン・九連石・斜め取り系・並び取り系など、日本のオセロ界で伝統的に使われている名称)を複数の情報源から調査する。**1つの情報源だけでなく、可能な限り複数の独立した情報源(戦術解説サイト、Wikipedia、オセロ関連ブログ・書籍紹介、大会結果解説など)を突き合わせ、手順・名称の食い違いがないか確認すること**(T009のFFOデータ収集で行ったクロスチェックと同様の手法)。
2. **自分の記憶だけで手順や定石名を捏造してはいけない**。必ずWebSearch等で実際に確認できた情報のみを収録し、各エントリに出典(URL等)を記載する。
3. 最低20種類以上の名前付き定石ラインを収集する(時間に余裕があれば50種類以上を目指してよいが、必須ではない。**質(正確性・出典の裏付け)を量より優先すること**)。
4. 各定石ラインについて以下の情報を記録する:
   - 定石名(例: "兎", "牛", "虎" 等。カタカナ・ひらがな・漢字の表記揺れがあれば主要な表記を採用し、別名があれば併記)
   - 手順(標準記法の着手列。例: `f5d6c3d3c4f4f6e3g5e6...`。着手ごとにどちらの手番か分かるよう、初手は黒固定という標準オセロのルールに従う)
   - 可能であれば分岐(同じ定石名で複数の変化がある場合はそれぞれ記録する。または、ある局面から複数の定石名に分岐する場合の分岐点も分かるように記録する)
   - 出典(URL、書籍名など)
   - 深さ(手数)
5. **序盤の初手について**: オセロは対称性により、初手は4通り(f5, d3, c4, e6相当)あるが、盤の回転・鏡映で互いに変換可能なため、実質1通りに正規化して扱うのが一般的(設計書もこの方針)。調査する際、情報源によって初手の表記が異なる場合(例: f5基準かe6基準か)があるので、収集時に注記しておくこと(正規化変換ロジック自体は後続タスクで実装するので、本タスクでは「元の情報源での初手表記」をそのまま記録し、備考欄に基準となる初手を明記すればよい)。
6. データの構造例(実装者の判断で調整してよいが、最低限これらの情報を持つこと):
   ```json
   {
     "lines": [
       {
         "name": "兎",
         "aliases": [],
         "moves": ["f5", "d6", "c3", "d3", "c4", ...],
         "firstMoveBasis": "f5",
         "sources": ["https://...", "..."],
         "notes": "..."
       }
     ]
   }
   ```

2026-07-08 implementer:

**実施内容**: WebSearch/WebFetchを用い、日本語オセロ定石(兎・虎・牛・ローズ系等)を公開情報源から調査し、`bookgen/joseki-research.json` を新規作成した。オセロクエストのアプリ・データベースは方針に従い意図的に情報源から除外し、個人運営の戦術解説サイト・定石データベースサイトのみを参照した。

**調査した情報源**:
- bassy84.net(「オセロ・リバーシの勝ち方、必勝法」): 虎・兎・牛・ローズ(シャープ/フラット/手塚システム)・こうもり・戦車・野ウサギ等、多数の個別定石ページをWebFetchで直接取得し座標列を抽出。
- hasera.net/othello/jouseki-tate.html: 虎・兎・牛系21ラインの名称と座標を一覧表形式で取得。
- hasera.net/othello/saizen40.html および othlog.hasera.net/jouseki/: 兎・虎の基本手順、および定石分類体系(縦取り系/斜め取り系等)のクロスチェックに使用。
- shoginoiroha.com, uguisu.skr.jp: 定石名称・分類の裏付け(座標なしの定性的確認)。

**収集結果**: 30種類の名前付き定石ラインを収集(要件の最低20種類を超過)。全エントリに出典URLを記載。うち5ライン(虎/兎/牛/野兎/ローズ基本形)について、2つ以上の独立ドメイン(bassy84.net, hasera.net, shoginoiroha.com)で着手座標が一致することを確認し、`methodology.crossChecks` に検証記録として残した。単一ソースのみのエントリ(短い分岐ライン等)には `notes` 欄に「単一ソース」である旨を明記した。

**検証**: `python -m json.tool bookgen/joseki-research.json` でパースエラーなしを確認。`json.load` で `lines` 配列が30件、全エントリに `sources` が空でないことをスクリプトで確認済み。

**スコープ外事項の確認**: DAG構造・8対称正規化・アプリ組み込み・SRS実装・エンジンでの手順検証は本タスクでは実施していない(T017以降に委ねる)。
