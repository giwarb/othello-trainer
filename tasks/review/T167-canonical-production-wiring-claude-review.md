`9ce0cc054b67807641b759a2e881a87dd562146dee5e4d659bba1efa228f54a4` で、タスクファイル記載の t166Manifest 値と一致(レビュー時に独立再計測)。

### 2. test-pattern-v5-wasm.mjs の実効性 — 形骸ではない

- PWV6 マジックバイト不一致 → `throw`(取り違えで確実に fail)

### 3. regression.rs のスキップ修正 — 弱めすぎではない

- `CARGO_MANIFEST_DIR/data/WTH_2000.wtb` が読めた場合はこれまで通りフル実行される構造(early return はファイル読取失敗時のみ)。WTHOR データのあるローカル環境では実行され続ける。
- CI では常にスキップ=このテスト単体は CI で実質無効化されるが、これはデータをコミットできないライセンス制約の帰結であり、代替はない。canonical の D4 不変性そのものは engine クレートの合成盤面プロパティテストが CI で担保しており、本テストは「実データでの裏付け」という追加保険の位置づけ。`train/tests/real_data.rs` の確立済みパターンの踏襲でもあり、妥当。
- 作業ログに「ファイル退避→スキップ確認→復元→通常実行確認」の両経路検証が記録されている。

### 4. worker.ts ロールバックコメント・cache.ts 版数 — 正確

- `ANALYSIS_ENGINE_VERSION` は 6 → 7(diff で前値 6 を確認)。cache.ts 側コメントの「6 -> 7」記載と一致。
- worker.ts の切り戻し手順は「(1) ファイル名を v4 へ戻す (2) 版数を必ず+1(7 から戻すなら 8、6 に戻さない・単調増加)」の 2 点セットとして明記されており、旧コメントの「ファイル名を戻すだけでよい」という誤誘導文が除去されている。T122 申し送りの解消として適切。worker.ts と cache.ts の双方向参照も張られている。

### 5. 旧 v4 経路の残置ファイル — 問題なし

- `app/public/` に pattern_v2/v3/v4.bin が残置。worker.ts からの参照はゼロ(grep 確認)で、タスクの「既存 pattern_v4.bin は残す(切り戻し用)」の指示どおり。
- sw.js はパターン bin をハードコードしておらず、プレキャッシュ対象は index.html 由来の静的アセットのみ。pattern_v5.bin は新 URL なので旧キャッシュにヒットせず、`CACHE_VERSION`(ビルドごとにコミットハッシュ注入)の activate 時に旧キャッシュ世代は削除される。stale 配信の経路はない。

## 指摘一覧

### 重大: なし

### 中

- **[中-1] `test-node-budget-wasm.mjs` が pattern_v4.bin のまま**(`app/scripts/test-node-budget-wasm.mjs:6`)。本番重みが v5 に切り替わった結果、時間バジェット系のビルドゲートだけが非本番重み(v4)で検証される非対称が生じた。v5 は scalar 特徴の評価コストが加わるため、バジェット挙動の検証は本番構成で行うほうが望ましい。今回のタスクスコープ(配線)外ではあるが、次回の軽タスクで v5 参照へ更新することを推奨(更新時は同スクリプトの期待値・閾値の妥当性も再確認)。

### 軽微

- **[軽微-1] ゲートの参照ファイルが worker.ts と独立にハードコード**。`test-pattern-v5-wasm.mjs` は `public/pattern_v5.bin` を直接読むため、将来 worker.ts の URL だけ変更(例: v4 へ切り戻し、v6 へ更新)してもゲートは旧 v5 ファイルを検証し続け、乖離を検知しない。現状は正しく一致しているので実害なし。将来 v6 化する際にスクリプト名・参照の更新を忘れない運用注意。
- **[軽微-2] ネイティブ評価との数値一致はゲート内でアサートされていない**。スクリプトは WASM 側の discDiff を JSON 出力するのみで、「全5局面で完全一致(0, -8.97, -2.33, 10.48, -1.57)」の照合は実装時の手動確認。期待値5個を定数としてスクリプトに埋めれば回帰ゲートに昇格できる(重みはgit管理下なので優先度低)。
- **[軽微-3] スクリプト内コメントが実在しない `compare-pattern-v5-native.mjs` に言及**(「相当の手順で行う」)。手動手順の意図は読み取れるが、実在ファイルと誤読しうる表現。
- **[軽微-4] regression.rs の `Err(_)` が全 IO エラーを飲む**。NotFound 以外(権限エラー等)でも無言スキップになる。また Rust の early-return スキップは CI 上「passed」と表示されるため、スキップ実態はログの eprintln でしか見えない。いずれも `real_data.rs` の既存パターンと同一挙動であり、本修正固有の劣化ではない。
