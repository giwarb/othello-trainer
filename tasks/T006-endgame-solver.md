# T006: 終盤完全読みソルバー

## 目的
空きマスが少ない終盤局面で、最終石差を完全に読み切る(近似ではなく真の最善手・真の石差を計算する)ソルバーを実装する。設計書のフェーズ1完了条件である「FFO endgame test 全問正解」を満たすために必須のモジュール。

## 背景・コンテキスト
- T002(bitboard)・T003(Zobrist/TT)完了が前提。T004(eval)とは独立なので並行して着手してよい。
- 設計書 §2.5.3「終盤ソルバー(完全読み)」を参照:
  - 空きマス24以下で石差完全読み、26以下で勝敗判定読み(本タスクでは石差完全読みのみ実装すればよい。勝敗判定読み(WLD: Win/Lose/Draw、石差を問わず勝敗のみを返す高速版)は余裕があれば実装してもよいが必須ではない。スコープ外に明記する)
  - 空き8以下: 奇偶(parity)順序付け + 隅優先
  - 空き4/3/2/1: 展開済み専用関数(関数呼び出し・ループを排したハードコード)— **本タスクでは必須要件にはしない**(あると探索が速くなるが、正しさより速度の最適化なので、まずは統一的な再帰関数で正しく実装することを優先する。時間に余裕があれば実装してよい)
  - 安定石による静的カット、NWS(null window search)主体 + TT
- 完全読みは「評価関数を使わず、終局まで読み切って実際の最終石差を返す」探索。評価値のスケールはT004と同じ**centi-disc(1石=100)ではなく、終盤ソルバーは整数の石差(1石=1)をそのまま返してよい**。ただしT005から呼び出す際にT004のスケール(centi-disc)に変換して統合する必要があるので、変換ルール(石差×100)をコメントで明記すること。
- パス処理: 手番側に合法手がなければパスして相手番。両者パスなら終局(残り空きマスは、その時点で多い方の色が総取りするルールを適用: 一般的なオセロの完全読みでは「終局時に空きマスが残っていても、それは石数が多い方に加算される」という慣習があるので、これに従うこと。設計書に明記はないが、標準的なオセロの完全読みソルバーの慣習として実装すること)。

## 変更対象(新規作成)
- `engine/src/endgame.rs` — 終盤完全読みソルバー
- `engine/src/lib.rs` — `mod endgame;` を追加

## 要件
1. `pub fn solve_exact(board: &Board, side_to_move: Side, tt: &mut TranspositionTable) -> i32` を実装する。返り値は「`side_to_move` から見た最終石差」(正なら勝ち、負なら負け、0なら引き分け)。T003の `TranspositionTable`/`zobrist_hash` を使い、置換表で枝刈りする(NWS: null window search + TTによるfail-soft alpha-betaを推奨)。
2. 探索の骨格は negamax + alpha-beta(またはNegaScout)でよい。空きマス数に応じた着手順序付け(parity考慮、隅優先)は「空き8以下」に限定せず、全体に適用してよい(実装をシンプルにするため。パフォーマンス要件は下記ベンチで担保する)。
3. 安定石による早期カット(例: 双方の安定石数から理論的に結果が確定する場合に探索を打ち切る)は**必須ではない**。実装できればより高速になるが、まずは正しさを最優先する。
4. パス・終局処理は上記の慣習(空きマスは終局時に多い方が総取り)に従うこと。両者合法手なしで終局した場合の最終石差計算にこのルールを適用する。
5. 空きマスが0の場合(終局)は即座に `black_count - white_count`(side視点に変換)を返す。
6. 単体テストで以下を検証する:
   - 空きマス1〜6程度の人工的な小さい局面をいくつか手動で構築し(または初期局面から数手進めて空きマスを減らした局面を使い)、`solve_exact` が返す石差が、**別の独立した実装(例: 深さ制限なしの単純な再帰関数を`#[cfg(test)]`内に別途書く、または全合法手を手動列挙して最善を確認する)と一致する**ことを検証する。
   - 手番を変えて同じ局面を解いたとき、`solve_exact(board, Black, ..) == -solve_exact(board, White, ..)` となること(手番以外の条件が同じ場合)。
   - 空きマス数が10〜14程度の局面でも数秒以内(目安。後述のベンチで正式計測するので、単体テストでは「タイムアウトしない」程度の緩い確認でよい)に完了すること。
