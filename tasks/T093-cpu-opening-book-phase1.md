# T093: CPU対局の定石ブック on/off(フェーズ1)

## 目的

ユーザー要望(2026-07-14)「自作エンジンにも opening book の on/off 2モードを用意してほしい」のフェーズ1。**既存の定石DB**(兎・牛・虎・酉等、T016/T017/T025で独自構築済み)を CPU の着手選択に接続し、対局モードのCPU設定に「定石ブック」トグルを追加する。Edax評価値付きの本格ブック生成(フェーズ2)は T090 基盤完成後の別タスク。

## 絶対制約(現在の環境事情)

**`engine/` `train/` のRustコード変更・cargoビルドを一切行わないこと。** 現在ローカルで長時間の教師コーパス生成(8プロセス)が実行中で、train クレートのバイナリ差し替えが生成を壊すため。本タスクは **app/ の TypeScript のみ**で完結させる(`npm run build` の prebuild が wasm ビルドを走らせるのは許容 — wasm32 ターゲットで別成果物のため干渉しない)。もしエンジン側の変更が必要という結論になったら、実装せず停止して報告せよ。

## 背景・コンテキスト

- 定石DB: `app/src/joseki/`(DAG+8対称正規化ハッシュ、T017)。定石練習モードには「相手が定石内を重み付きランダムで進行する」ロジックが既に存在する(T020)。これを対局モードのCPU着手に再利用する。
- 対局モードのCPU着手: `app/src/app.tsx` の CPU 着手リクエスト(T085cで強CPUは maxNodes:160000)。
- 参考UX: オセロクエスト式(相手が重み付きランダムで定石内進行、外れたら通常思考)。

## 要件

1. **ブックon時のCPU着手**: CPUの手番で現局面(8対称正規化)が定石DB内にあり後続手が存在する場合、**エンジン探索を呼ばずに**定石の後続手から重み付きランダムで即座に着手する。定石を外れた(DBに無い/終端に達した)以降は従来どおりエンジン探索。
2. **ランダム性と決定性**: ブック着手の乱数は `Math.random` でよい(対局の多様性が目的。エンジン探索自体の決定性は不変)。
3. **トグルUI**: 対局モードのCPU設定(強さ選択の近く)に「定石ブック」on/offトグルを追加。**既定はon**。設定はlocalStorageに永続化。全CPU強さに適用(弱いCPUも定石内は定石を打つ)。
4. **表示**: ブック由来の着手は、既存の評価ソース色分けの「定石」表示(joseki source)と整合させる(评価バッジがある文脈では「定石」と示す。新しい表示システムは作らない)。
5. 人間側の着手には影響しない(悪手判定・オーバーレイ等は従来どおり)。
6. テスト: 正規化ハッシュでのブックヒット判定・重み付き選択・外れ後のフォールバックの単体テスト。既存テスト(app 510件)を壊さない。
